1. SQL injection  공격
    - 개념 : DB와 연동되  앱에 입력을 조작하여 의도치 않은 결과를 반환시킴
    - 전송된 입력값에 적정성 검사가 없다면 발생

    종류 :
        - Form SQL injection : html form 기반 인증을  담당하는 애플리케이션에 취약점이 있을 때 쿼리문 조건을 임의로 조작하여 인증을 우회
        - Union SQL injection : union selet 를 활용하여 다른 쿼리의 결과를 결합하여 도출
        - ERROR-BASED SQL injection : DB 쿼리에 대한 에러값을 기반으로 DB 정보 획득
        - Blind SQL injection : ERROR-BASED와 달리 쿼리 결과의 참과 거짓을 통해 SQL문을 실행하여 데이터 베이스를 비정상적으로 공격
    
    대책 :
        입력에 특수 문자가 있는지 검증
        SQL 서버의 에러 메세지 미표시
        일반 사용자 권한으로 시세틈 저장 프로시저 접근 불허

2. CSS
    - 개념 : 공격자의 작성된 스크립트가 다른 사용자에게 전달
    - 브라우저 내에서 적절한 검증 없이 실행되기 때문에 사용자의 세션을 탈취 또는 사이트 변조 또는 악의적 사이트로 사용자를 이동시킴

    종류 :
        - Stored XSS : 가장 일반적인 공격 사용자가 글을 저장할 수 있는 부분에 정상적인 평문이 아닌 스크립트 코드 입력
        - Reflected XSS : 스크립트가 포함된 공격  url 을 다른 웹사이트나 이메일 등으 로탱 보내서 다른 사용자가 그것을 클릭하도록 유도
        - DOM based XSS : 피해자 브라우저가 html 페이지를 구문 분석할때마다 공격  스크립트가 dom 생성의 일부로 실행되면서 공격, 페이지 자체는 변하지 않은나; 페이지에 포함된 브라우저 측 코드가 돔 환경에서 악성코드로 실행됨
    
    대책 :
         입력한 문자열의 <>&""등을 문자 변환 함수나 메소드 replace all() &lt, &gt, &amp, &quot로 치환
         지원하는 html  태그의 리스트를 선정한 후 해당 태그만 허용하는 방식을 적용

3. CSRF
    - 개념 : 특정 사이트에 대해 사용자가 인지 하지 못한 상황에서 사용자 의도와 다르게 무관하게 공격자가 의동한 행을 요청하게 되는 공격
    - XSS와의 차이는 XSS는 클라이언트에서 실행 CSRF는 스크립트가 서버에서 요청한다는 차이가 있음

    대책 : 입력화면 폼 작성시 GET방식보다 POST 방식을 사용하고 입력호면 폼과 해당 입력을 처리하는 프로그램 사이에
    토큰을 사용하여 공격자의 직접적인 URL 사용이 동작하지 않도록 처리한다, 사용자 세션 검증, 재인증을 유도

4. 업로드 
    규제가 없다면 악용 웹셀 관리자 권한 획득하여 원격실행 명령을 수행함, 파일 확장자 검증 필요

5. 다운로드
    상위 디렉터로 탐색 득정 시스템 파일 다운로드 .. 와 같은 문자열 필터링

6. 쿠키 세션 위조 
    서버 사이트 세션 사용 및 쿠지 정보 암호화
